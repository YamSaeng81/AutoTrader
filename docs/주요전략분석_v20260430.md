## 1. 공통 구조 — 가중 투표 엔진

[CompositeStrategy.java](../core-engine/src/main/java/com/cryptoautotrader/core/selector/CompositeStrategy.java)

```
buyScore  = Σ(weight × confidence)   BUY 신호 합산
sellScore = Σ(weight × confidence)   SELL 신호 합산
normalizer = max(totalWeight, 1.0)   → 정규화

상충: buyScore > 0.4 AND sellScore > 0.4 → HOLD
강한: > 0.6 → STRONG BUY/SELL
약한: > 0.4 → BUY/SELL
미달: ≤ 0.4 → HOLD
```

### 2.2 알려진 구조적 결함

> ⚠️ 이 전략에는 현재 두 가지 설계 결함이 존재한다. P1 코드 개선 예정.

**결함 1 — EMA 이중 카운팅**
- EMA 방향 필터: EMA(20) vs EMA(50)
- 하위 전략 EMA_CROSS: EMA(20) vs EMA(50)
- **동일한 두 EMA 값을 두 군데서 따로 사용**. 골든크로스 발생 시 EMA_CROSS BUY(0.1 기여) + EMA 방향 필터 통과로 EMA의 실질 영향력이 0.1을 초과함.
- P1 개선: EMA_CROSS(0.1) → MACD(0.1) 교체 (독립 모멘텀 신호)

**결함 2 — RSI(0.2) 수학적 무의미**
- 현재 구조에서 RSI 단독 sellScore > 0.4 만들려면 confidence > 2.0 필요 (불가).
- RSI는 점수를 미세하게 흔드는 역할만 하며, 과매수 구간 진입 자체를 막지 못함.
- P1 개선: RSI를 가중치 투표에서 제거 → **RSI Veto Gate**로 전환 (RSI > 75이면 BUY 강제 차단)

**결함 3 — ADX 임계값 고정의 실전 문제**
- 2026-04-27 핫픽스 기록: BREAKOUT 세션 4개(SOL/XRP/ETH/BTC)가 ADX(16~18)<20으로 **100% 차단** → 임시 완화(15.0).
- 고정 임계값이 시장 상황에 따라 전략을 완전히 죽이거나 살림.
- P1 개선: ADX 30일 30th percentile 기반 동적 임계값

### 2.3 신호 흐름

```
[1] ADX 횡보 필터   ADX(14) < threshold → 즉시 HOLD
[2] 가중 투표       buyScore / sellScore 계산
[3] 최종 신호 판정  > 0.6 강 / > 0.4 약 / 상충 HOLD
[4] EMA 방향 필터   추세 역행 → HOLD
```

### 3.3 Ichimoku 구름 필터 — 재평가

**⚠️ 중요: Ichimoku 필터는 위험관리 도구가 아니다.**

|---|---:|---:|

MDD가 동일하다 = Ichimoku 필터가 최악 손실 구간을 전혀 막지 못했다.
추세가 확립된 하락장(가격 구름 아래)에서는 필터가 작동하지 않기 때문이다.
**Ichimoku 필터의 실제 기능: 추세 전환 구간(구름 내부)의 저품질 신호 제거 — 수익률 소폭 개선 / 위험 미감소.**

EMA 방향 필터 vs Ichimoku 필터 역할:
- EMA 방향 필터: *추세 방향 역행* 신호 차단
- Ichimoku 구름: *추세 방향 자체가 불확실한 구간* 차단

두 필터는 보완적이나, **둘 다 크래시 방어 기능은 없다.**

### 4.2 SUPERTREND 동작

```
ATR(10) 기반 동적 추세선
상단밴드 = 중간선 + multiplier × ATR(10)
하단밴드 = 중간선 - multiplier × ATR(10)

가격 > 직전 상단밴드 → 상승 추세 확립 → BUY
가격 < 직전 하단밴드 → 하락 추세 확립 → SELL
밴드 내부           → 추세 유지 (이전 신호 지속)
```

**MACD와 시너지**: MACD는 크로스로 전환 포착, SUPERTREND는 밴드 돌파로 유지 검증 → 방향 동의 시 상충 거의 없음.
